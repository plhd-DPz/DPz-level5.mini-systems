# Mini Systems

This repository contains structured multi-file applications and small-scale systems built to simulate real-world project development.

## Purpose

- Practice modular design  
- Organize multi-file projects  
- Manage resources and assets  
- Improve maintainability  
- Transition from small apps to real systems  

## What qualifies as Level 5

- Multiple source files  
- Clear folder structure  
- Separated logic and resources  
- Defined application flow  
- Feature-complete mini applications  

## Typical Examples

- Mini games  
- CLI tools with modular structure  
- Optimization utilities  
- Small-scale management systems  

## Structure

project_name/
    src/
    resources/
    main_entry
    README.md

## Focus Areas

- Code separation  
- Basic architecture thinking  
- Refactoring  
- Reusability  
- Project organization  

## Philosophy

Not just writing code.  
Designing small systems with intention.
